About This Data Set

Is the data complete?

No, the data is not complete.

There are a few problems impacting the data's completeness.

  • First and most simply, twice my phone's battery depleted and the phone turned off, which means data is missing from those times.
  • Because the AWARE framework takes up a lot of storage space, and I'm not sure if my backup sync was always working properly, sometimes the data took up so much space that my phone stopped storing new data (not only for AWARE, but also for other apps). This may have lead to some missing data.
  • When the storage was maxed out, I tried to sync the existing data and then delete the backup on the phone itself. I do not know if that may have deleted some data that was not backed up.
  • The data from some sensors and plugins that were turned on (like Google Fused Locations) never showed up in the database.

Is the data coherent?

The first thing that did not seem internally coherent was times -- they were off by a few hours from what would seem like logical times that I'd be using my phone. I realized that the times were stored in GMT by default and needed to be converted to EST, which then made more sense.

Other than that the data seems to be coherent.

Is the data correct?

I am mostly only able to verify the data through my own recollections and general behavior patterns. The data seems to be showing phone use and charing around times that would make sense.

A few times during the data collection when I checked the database I would spot check the most recent entry in a certain table. Aside from the missing data issues, those times and sensed data seemed general correct against what had happened just before the sync.
